# Obsidian Vault

このリポジトリはObsidian Vault。Markdownノートの作成・編集・整理を行う。

## 構成

- `00_Profile/` — ユーザーのプロフィール・価値観・目標・AI応答の好み。他の作業の前提として参照する。
- `01_Projects/` — 進行中プロジェクトごとのノート。
- `02_Knowledge/` — 分野別の知識ベース（Marketing / AI / Python / Finance）。
- `03_Daily/` — デイリーノート。ファイル名は `YYYY-MM-DD.md`。
- `04_Ideas/` — 未整理のアイデア・メモ。
- `99_Archive/` — 完了・非アクティブになった内容の保管場所。

## 運用ルール

- 新しいプロジェクトは `01_Projects/` にサブフォルダを作る。
- 分野知識は `02_Knowledge/` の該当フォルダに追記していく。
- ノート内リンクはObsidianの `[[ノート名]]` 記法を使う。
- Obsidian内部リンクを壊さないよう、ファイル移動・リネーム時は参照元も確認する。

## Profile参照ルール（必要時のみ）

`00_Profile/`（Profile.md / Preferences.md / Values.md / Goals.md）は、毎回自動では読み込まない（トークン消費を抑えるため）。

以下のようなケースでは、ユーザー背景・価値観・好む回答スタイルの把握が必要と判断し、読み込む前提でユーザーに一言確認する。

- 職歴・専門領域を踏まえた助言や壁打ちが必要な相談（マーケティング、キャリア、事業戦略など）
- 趣味関連の作業（カメラ・キャンプ・コーヒーなど）で好みや所有ギアの前提が要る場合
- 目標・価値観に照らした意思決定支援（投資・不動産など）
- ユーザーから「プロフィール踏まえて」「いつもの感じで」等の明示的な指示があった場合

単純な事実確認・雑談・ツール操作のみのタスクでは読み込まない。
